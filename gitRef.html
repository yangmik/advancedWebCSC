Learning Git and Github - My Reference Sheet

Navigation Commands

To list your local directory, first use the ls or dir command at the Git Bash. In my Windows environment, both of these commands listed my directories.

To navigate into these folders, you will need to use the cd command followed by the name of the folder. In my case, I had to navigate into the Program Files (x86) directory. Because the folder has spaces in its name, the quotation marks are necessary. The command I used was: cd "program files (x86)"

Initializing a stand-alone, local Git Repository

Once you are in the correct directory using the git init command at the Git Bash allows you to initialize your local folder for Git. On my command line in Git Bash, I typed in: git init

Once you local folder is initialized, use the git add . command in the bash to prepare your local files for upload to the remote repository. The period selects all files in that initialized folder.

Next, use git commit -m"Initial Commit" at the Git Bash and then type git remote add origin https://github.com/your account name/name of repository. This will tell Git where remotely your files will need to copy to.

The last command is to upload the data to your remote repository. Type the command git push -u origin master and watch your local files upload to the repository.

How to get the most recent code from Github

If you do not have the current remote repository on your local machine, you will need to browse using Git Bash to the folder you'd like to stage. Then you can use the command git clone at the Git Bash to initialize your local folder for Git. On my command line in Git Bash, I typed in: git clone https://github.com/URL to repository

The URL of your repository is shown in the repository on GitHub.com.

If your local folder is ready, you can use the command git pull https://github.com/USERNAME/Repository.git and the most recent files will be pulled to your local machine for you to work on.

Adding commit changed to a Repository

Once the files on your local machine are ready to be committed to the remote repository, you must add these changed files to the staging area by using the command git add . (remember the period in the command adds all changed files) followed by the command git commit -m"Added Second comment". The last step is to use the command git push at the Git Bash to add the changes to those documents to the remote repository.

How to check the status of the Repository

Using the command git status at the Git Bash will allow you to see the current status of the Repository. This is an important command as it lets you know if your repository is up to date or if its out of date. The command will check to make sure that the branch repository, in this case, the repository on your local machine is current with the remote repository or Origin/Master repository. It will also check to make sure if there are files ready to commit.

How to save your changes to a repository on GitHub

The git add command tracks the changes to the file and stages it for transfer. These files are not committed until you use the git commit and the git push commands.

How to view a list of all the commits in a Repository

The git log command will list the commits in a Repository and is a useful tool to see who has made changes to your repository.

How to make a temporary branch from the master Repository

The git branch command will allow you to list, create, or delete branches in the repository. Branching the repository allows you to keep the main code in tact while branching or copying the repository to a branch repository and allowing you to make changes necessary. If that branch and those changes are not needed, they can be deleted and reverted back to the original version before the branching occurred..

How to merge a branch back into the master Repository

Let's say you are working on fixes on the code in a branch repository and eventually it needs to be merged back to the original code, the git merge command will allow you to merge your branch to the original repository.
